Dividend Uploader 项目说明

网站后台配置 + MT4/MT5 服务器插件远程同步 JSON。本文档是当前项目唯一主说明，所有 Markdown 文档已经合并到这里。

1. 项目目标

当前方向已经确定：不走 Manager API 开发路线，改为 MT4/MT5 Server 插件 + 远程网站后台更新 JSON 配置。

  • 后台网站负责登录、配置、保存版本、发布当前生效 JSON。
  • MT4/MT5 插件定时访问网站公开接口，读取当前生效配置。
  • 股息上传和隔夜利息数值更新都在网站页面选择平台、服务器、生效时间后执行配置。
  • 隔夜利息只更新数值，不需要每次修改收取模式或三倍日。
  • 当前简化为 CSV 导入、导出和页面内编辑，不依赖 Finhub 自动同步。
  • 插件同步基础 URL 在后台页面配置，不限定为 test.appcdn002.com。
  • 生效时间按网站业务时区录入，同时发布 UTC 时间给插件统一判断；插件默认读取 MT4/MT5 服务器时区。
  • 升级包不覆盖已有配置、反馈记录、管理员账号和运行数据。

2. 最终目录结构

根目录只保留 4 个主要目录，其他网站源码、运行数据、安装脚本和可选工具都收进 website/。

Dividend Uploader/ ├─ index.html 项目唯一说明文档 ├─ package.json 本地运行、测试、打包命令 ├─ VERSION 当前版本号 ├─ website/ 网站后台完整工程 │ ├─ src/ 网站程序源码 │ ├─ config/ 导入模板和默认配置 │ ├─ install/ Linux 一键安装脚本 │ ├─ runtime/ 本地运行数据，不进升级包 │ └─ optional/ 非必须工具、测试、临时包 ├─ plugins/ MT4 / MT5 插件工程 │ ├─ mt4/ │ └─ mt5/ ├─ sdk/ MT4 / MT5 SDK 放置目录 │ ├─ mt4/ │ └─ mt5/ └─ packages/ 最终安装包、升级包、GitHub 包

3. 必须与非必须

必须保留

位置用途

4. GitHub 上传安装

  1. 打开 packages/github-website/，找到最新时间目录。
  2. 上传其中的 DividendUploader_GITHUB_WEBSITE_v版本.zip 到 GitHub 仓库。
  3. 在 Linux 服务器拉取或解压 GitHub 仓库。
  4. 进入项目根目录，运行 sudo bash website/install/install-linux.sh。
  5. 按提示输入域名 test.appcdn002.com、端口、管理员账号和密码。

不要把 website/runtime/、.env、管理员密码和真实运行数据上传到 GitHub。

5. 一键安装脚本

安装脚本位置：

website/install/install-linux.sh

在 Linux 服务器项目根目录执行：

sudo bash website/install/install-linux.sh

脚本会完成：

  • 检查或安装 Node.js。
  • 复制网站文件到安装目录。
  • 创建独立运行数据目录，避免升级覆盖配置。
  • 写入网站端口、管理员账号、后台密钥等配置。
  • 创建 systemd 服务并启动网站。
  • 可选配置 Nginx 反向代理到 test.appcdn002.com。

6. 后台使用方法

  1. 打开 https://test.appcdn002.com。
  2. 使用安装时设置的管理员账号登录。
  3. 日常操作进入 CSV Data：选择 CSV 类型，导入 CSV，或直接在表格里新增、修改、删除行。
  4. 需要保留文件时，点击 Export CSV 导出当前页面数据。
  5. 不常变化的设置进入 Configuration：左侧选择 Products、Environment、Execution、Remote Sync，右侧用标签页切换子设置。
  6. 产品较少变化，统一在 Configuration / Products 维护，也可以导入或导出产品 CSV。
  7. 平台、服务器、生效时间在 Configuration / Environment / Target 选择。
  8. 网站业务时区和插件时区来源在 Configuration / Environment / Time Zone 选择；插件默认是 Auto: MT4/MT5 server timezone。
  9. 插件同步基础 URL 和配置文件名在 Configuration / Remote Sync 设置，可使用正式站、测试站或内网地址。
  10. 点击 Apply To JSON 把 CSV 行写入待发布 JSON，再到 Plugin Sync 保存并发布为当前生效配置。
  11. 页面右上角只显示插件连接状态，用小圆点表示等待、连接成功、错误或过期。

7. 插件与 SDK

SDK 放置位置

  • MT4 SDK 放到 sdk/mt4/MT4_API/。
  • MT5 SDK 放到 sdk/mt5/MetaTrader5SDK/。

Visual Studio 工程

  • MT4：打开 plugins/mt4/DividendUploaderMT4RemoteConfigPlugin.sln。
  • MT5：打开 plugins/mt5/DividendUploaderMT5RemoteConfigPlugin.sln。
  • 选择 Release，按服务器要求选择 Win32 或 x64，生成 DLL。
  • 输出目录默认在 packages/plugin-build/。

插件 URL 与时间配置

  • 插件工程中有 RemoteSyncSettings，包含 base_url、公开读取路径、反馈路径和轮询文件名。
  • 正式接入 SDK 时，把这些字段接到插件参数、INI/JSON 配置文件或服务器插件配置里。
  • 插件读取 updated_at 和 targetContext.effectiveFromUtc 时按 UTC 判断，显示和人工确认按 timeSync.businessTimezone。
  • timeSync.serverTimezone 默认为 auto-mt-server，表示插件从当前 MT4/MT5 服务器读取服务器时区；只有后台手动选择具体时区时才覆盖。

当前插件工程已经准备好项目文件，但真正编译 DLL 还需要你放入 MetaQuotes 对应版本的 Server Plugin SDK。

8. 插件公开接口

插件不需要登录后台，只访问公开读取接口和反馈接口。

接口用途
/admin-api/dividend-uploader-public/active-meta.json读取当前配置 UUID、版本、更新时间。插件可先比较 UUID，避免重复下载。
/admin-api/dividend-uploader-public/active.json读取当前完整生效配置。
/admin-api/dividend-uploader-feedback插件用 GET 上报心跳、应用结果、错误信息。

上面的路径是默认路径；完整 URL 由后台配置里的 remoteSync.baseUrl 拼接生成，所以可以使用任意正式域名、测试域名或内网地址。

9. 打包输出

在项目根目录执行：

npm run package:all

会生成：

  • packages/github-website/：上传 GitHub 用的网站包。
  • packages/plugin-config-portal/：网站安装包、升级包、MT4 插件工程包、MT5 插件工程包。
  • website/optional/packages/simple-run/：Windows 简单运行测试包。

10. 测试与验证

本机快速打开页面

python3 website/optional/local-test/run_local.py

如果检测不到 Node.js，脚本会自动尝试安装：Windows 优先使用 winget，其次使用 choco；macOS 会尝试使用 Homebrew。

只准备测试数据，不打开浏览器

python3 website/optional/local-test/run_local.py --prepare-only

运行自动测试

npm test

如果在 Windows 双击没有反应，可以先打开命令行进入解压目录，再运行：

py run_local.py

如果你不希望脚本自动安装 Node.js，可以加 --no-auto-install。
