Dividend Uploader Windows 快速测试

当前主线：网站后台管理配置，MT4/MT5 插件远程同步 JSON，并更新系统隔夜利息 long/short 数值。

本地页面
运行 start-web.bat 自动加载测试环境并打开页面。
插件工程
MT4/MT5 各有独立 Visual Studio solution。
安装包
网站安装包、升级包和插件包统一输出到 packages。

1. 解压或复制到 Windows

建议使用简单路径：

D:\Plugin\DividendUploader

目录里应能看到：

  • run_local.py
  • start-web.bat
  • plugins\mt4\DividendUploaderMT4RemoteConfigPlugin.sln
  • plugins\mt5\DividendUploaderMT5RemoteConfigPlugin.sln
  • sdk\mt4\
  • sdk\mt5\

2. 启动本地页面

  1. 双击 check-runtime.bat，确认 Python 和 Node.js 能显示版本。
  2. 双击 start-web.bat。
  3. 脚本会自动加载测试数据并打开浏览器。

页面地址：

http://127.0.0.1:4173
不要从 Python IDLE 里运行 run_local.py。如果有错误，IDLE 可能看起来像“没反应”。

3. 生成交付包

在项目根目录打开 PowerShell 或命令提示符：

npm run package:all

输出目录：

packages\plugin-config-portal
packages\simple-run

需要交付的网站包和插件包都在这里。

4. 编译 MT4 插件 DLL

  1. 打开 plugins\mt4\DividendUploaderMT4RemoteConfigPlugin.sln。
  2. 通常先选择 Release | Win32。
  3. 确认官方 MT4 Server Plugin SDK 已放到 sdk\mt4\，或在项目属性里修改 MT4_SDK_DIR。
  4. Build 后到 packages\plugin-build\ 取 DLL。

当前 sdk\mt4\MT4_API 主要是文档，正式可加载插件需要补官方 MT4 Server Plugin SDK 的头文件和库文件。

5. 编译 MT5 插件 DLL

  1. 打开 plugins\mt5\DividendUploaderMT5RemoteConfigPlugin.sln。
  2. 选择 Release | x64。
  3. 默认 SDK 路径是 sdk\mt5\MetaTrader5SDK。
  4. Build 后到 packages\plugin-build\ 取 DLL。

真实隔夜利息更新实现优先参考：

sdk\mt5\MetaTrader5SDK\Examples\Server\InterestRates
sdk\mt5\MetaTrader5SDK\Examples\Server\SwapsCopier

6. 网站安装和插件同步

网站生产域名可以在后台页面配置，不限定测试域名。默认示例：

https://test.appcdn002.com

插件公开读取地址：

https://test.appcdn002.com/admin-api/dividend-uploader-public/active-meta.json
https://test.appcdn002.com/admin-api/dividend-uploader-public/active.json
https://test.appcdn002.com/admin-api/dividend-uploader-feedback

插件生效时间按页面选择的业务时区录入，发布给插件时同时带 UTC 时间。

完整部署步骤见：

index.html

7. 关键原则

  • 隔夜利息只更新系统 long/short interest 数值。
  • 不通过客户 Balance/Deal 实现隔夜利息。
  • 平台、服务器、生效时间在页面选择，不写入 CSV。
  • 计息模式和三倍日只读展示，不随 CSV 修改。
  • 升级网站不能覆盖已有配置、UUID、管理员账号和插件反馈记录。

8. 自动安装依赖

如果运行 run_local.py 时检测不到 Node.js，脚本会自动尝试安装 Node.js 24+。

  • Windows 优先使用 winget 自动安装。
  • 如果没有 winget，会尝试使用 choco。
  • 安装完成后脚本会自动查找常见 Node.js 安装目录，并继续启动页面。
  • 如果不想自动安装，可以运行 py run_local.py --no-auto-install。
